class farg.core.ui.gui.RunForNSteps(controller, gui, num_steps)[source]

Runs controller for up to n steps.

This does not update the GUI directly; It however results in changing the state of the attribute “controller” that it holds. This is shared by the GUI and used to update itself.

Before each step, checks that we have not been asked to pause.

RunForNSteps.controller = None

Controller for the whole app.

RunForNSteps.gui = None

GUI being displayed. We need this to communicate some state (such as “we have found an answer and can now quit.”).

RunForNSteps.num_steps = None

Number of steps taken so far.

UI — GUI — Central Pane¶

Central pane of the UI display.

This is basically a canvas that can hold multiple views (such as Coderack view, Workspace view, and so forth). Each view is an instance of a subclass of ViewPort.

UI — GUI — Views¶

UI — GUI — Views — Viewport¶

Viewport defines a section of a canvas for displaying some information.

The coordinates (left, top) and (bottom, right) define a rectangular section of the canvas.

This is an abstract class where a few methods need to be plugged in.

UI — GUI — Views — List based¶

Defines a class that can act as a base class for displaying lists of things.

Examples of such things include codelets in a coderack and recent foci in the stream.

Obtain a 3-tuple (items, top-message, extra_dict) to display.
