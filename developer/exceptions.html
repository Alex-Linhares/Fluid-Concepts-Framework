Exceptions¶

Exception classes in FARG core.

Note

Exceptions specific to an individual application should live in that application’s directory.

exception farg.core.exceptions.AnswerFoundException(answer, codelet_count)[source]¶

Raised by a subspace when it believes that an answer has been found.

exception farg.core.exceptions.BatchModeStopException(codelet_count)[source]¶

Base class of ways of stopping during batch mode.

Look at the subclasses (in this file) for details.

codelet_count = None¶

Number of codelets that had been run when the exception was raised.

exception farg.core.exceptions.FargError(msg='')[source]¶

Base class for untrappable errors (indicating bugs).

msg = None¶

Message to be displayed.

exception farg.core.exceptions.FargException[source]¶

Base class for FARG-specific exceptions.

exception farg.core.exceptions.NoAnswerException(codelet_count)[source]¶

Raised by a subspace when it is realized that no answer is forthcoming.

exception farg.core.exceptions.StoppingConditionMet(codelet_count)[source]¶

When a stopping condition is specified, this indicates that it has been reached.

exception farg.core.exceptions.SuccessfulCompletion(codelet_count)[source]¶

Raised when the problem has been fully solved.

What fully solved means depends on the application, of course. For Seqsee, this means currently means “Sequence has been extended to all known terms.”.
